# Murge — Daily Merge Game

A daily puzzle-merge game built in Unity 6 (6000.3.11f1) with Supabase backend. Targeting iOS and Android. Players drop balls, merge matching tiers, and compete on a daily leaderboard. Same seeded sequence for all players each day.

## Build & Deploy

```bash
# Supabase Edge Functions
cd Supabase && ./deploy.sh dev    # deploy to dev project
cd Supabase && ./deploy.sh prod   # deploy to prod project

# iOS
./build-ios.sh dev               # dev build → TestFlight
./build-ios.sh dev --no-upload   # build only, opens Xcode (auto-runs pod install)
./build-ios.sh prod              # prod build → TestFlight

# Android
./build-android.sh dev           # dev APK
./build-android.sh prod          # prod AAB (for Google Play)
```

Build number auto-increments from `.build-number`. The gameplay screen is built at **editor time** by `GameSceneBuilder.cs` — run **MergeGame > Build Game Scene** then Cmd+S after changes. Other screens self-build at runtime via `OnEnable`.

## Architecture

### Namespaces

- `MergeGame.Core` — GameManager, BallController, DropController, DailySeedManager, ScoreManager, MergeTracker
- `MergeGame.UI` — All screens and HUD (procedurally built, no prefabs)
- `MergeGame.Backend` — SupabaseClient, AuthManager, LeaderboardService, OfflineSyncQueue, NativeSignIn
- `MergeGame.Data` — ScriptableObjects and design tokens (BallData, PhysicsConfig, MurgeDesign)
- `MergeGame.Visual` — BallRenderer, WaveformAnimator, MergeParticles, PixelUIGenerator
- `MergeGame.Audio` — AudioManager (procedural SFX)
- `MergeGame.Editor` — GameSceneBuilder, BuildScript, iOSPostBuild

### Key Patterns

- **Singletons** everywhere (GameManager.Instance, ScoreManager.Instance, etc.)
- **Callback-based async** for network calls (no async/await)
- **Coroutines** for animations, timing, and settle detection
- **Procedural rendering** — ball sprites generated by BallRenderer, waveforms by shader

### Game Flow

```
App Launch → PreloadAndNavigate() → Check saved game?
  → Yes: ResumeGame() → Playing
  → No:  First launch? → Onboarding → HomeFresh
                       → HasPlayedToday? → HomePlayed / HomeFresh
Playing → Drop ball → Merge same tiers → Score with combo multiplier → Game Over → Results
```

### Daily Seed System

`DailySeedManager` generates a deterministic ball sequence from the date string hash (xorshift128). Same seed = same sequence for all players. Game state persistence saves/restores the sequence index.

## Design System

- Colors: `OC.cyan` (#4DD9C0), `OC.bg` (#0F1117), `OC.surface` (#161B24), `OC.border` (#232838), `OC.muted` (white 22%), `OC.amber`, `OC.pink`
- Fonts: `MurgeUI.PressStart2P` (headings/buttons), `MurgeUI.DMMono` (body text)
- Button height: **44px** everywhere. Primary: cyan fill + scanlines. Ghost: outline border + bg fill.
- Ball tiers: 11 total (index 0-10). Combo multiplier: `1.0 + (chainLength - 1) * 0.25`, capped at 2.5x.
- Only basic ASCII in UI text — PressStart2P lacks unicode glyphs (use `x` not `×`).

## Supabase Backend

### Edge Functions (in `Supabase/supabase/functions/`)

| Function | Purpose |
|----------|---------|
| `register-player` | Upsert player, generate Phish-themed default name |
| `submit-score` | Insert daily score with merge_counts + longest_chain |
| `get-leaderboard` | Ranked player list for a game date (public, no auth) |
| `get-player-rank` | Player's rank + total players |
| `get-player-profile` | Today's score, streaks, merge counts |
| `update-display-name` | Validate, sanitize, profanity check |
| `sync-streak` | Update streak counts |
| `migrate-player` | Transfer player data + scores between user_ids |

- `_shared/auth.ts` — JWT verification (all endpoints except get-leaderboard require Supabase JWT)
- `_shared/profanity.ts` — obscenity-based content filter
- Tables: `players` (user_id PK, display_name, streaks, auth_provider, auth_email), `daily_scores` (user_id + game_date unique, score, merge_counts, longest_chain)
- Score validation: hard cap 99,999, game_date within +/-1.5 days, longest_chain capped at 99
- Dev vs Prod: `SupabaseClient.cs` auto-selects credentials based on build type. Bundle IDs: `com.murge.game.dev` / `com.murge.game`.

## Authentication

Uses Supabase Auth (GoTrue REST API) via `AuthManager.cs`. No Supabase SDK — raw HTTP calls.

- **Anonymous auth**: Created when user taps "LET'S PLAY" in onboarding (`POST /auth/v1/signup`)
- **Apple Sign In** (iOS): Native plugin `AppleSignInPlugin.mm` → ASAuthorizationController
- **Google Sign In** (iOS): Native plugin `GoogleSignInPlugin.mm` → Google Sign-In SDK via CocoaPods
- **Google Sign In** (Android): Native plugin `GoogleSignInPlugin.java` → legacy play-services-auth
- **Token exchange**: ID tokens → Supabase session via `POST /auth/v1/token?grant_type=id_token`
- **Identity linking**: GoTrue REST API does NOT support linking ID tokens to anonymous users. Data-level migration instead: sign in with provider (new user_id), `migrate-player` transfers data, old anonymous user deleted.
- **Token refresh**: Automatic, 5 min before 1-hour expiry
- **Session persistence**: PlayerPrefs (access_token, refresh_token, user_id, provider)

```
Onboarding → "Sign in" → Apple/Google direct auth | "LET'S PLAY" → anonymous
Settings → Connect Account → provider sign-in → migrate-player → delete anonymous
Sign Out → clear local data → new anonymous session → fresh home
```

Auth key files: `Backend/AuthManager.cs`, `Backend/NativeSignIn.cs`, `Plugins/iOS/AppleSignInPlugin.mm`, `Plugins/iOS/GoogleSignInPlugin.mm`, `Plugins/Android/GoogleSignInPlugin.java`, `Plugins/Android/MurgeActivity.java`, `UI/SignInSheet.cs`, `UI/LinkPromptCard.cs`, `UI/SignOutConfirm.cs`, `Editor/iOSPostBuild.cs`, `scripts/generate-apple-secret.rb` (Apple JWT, expires every 180 days).

Google OAuth: 4 client IDs in Google Cloud Console (Web, iOS, Android prod, Android dev). Embedded in native plugins. Android clients keyed to package name + SHA-1.

## Persistence & Network

- Game state saved to PlayerPrefs after each drop settles + on app background. Discarded on day rollover or game over.
- `NetworkMonitor.cs`: connectivity via `Application.internetReachability` + HEAD ping. Polls every 30s offline.
- All POST calls retry 3x with exponential backoff (1s, 2s, 4s). 4xx errors skip retry.
- `OfflineSyncQueue` persists failed calls to PlayerPrefs, flushes when connectivity returns.
- Launch date defined once in `GameSession.LaunchDate`.

## Analytics (GameAnalytics)

SDK: `com.gameanalytics.sdk` via OpenUPM. Wrapper: `MurgeAnalytics.cs`. All events fail silently. Events: `puzzle:start/score/duration/highest_tier/total_merges/longest_chain`, `social:share_card`, `engagement:streak/name_changed`, `onboarding:complete`, `gameplay:shake_used`.

## Conventions

- No emoji in code or UI unless explicitly requested
- Prefer editing existing files over creating new ones
- All UI screens build themselves procedurally — no scene-based UI
- GameSceneBuilder changes require scene rebuild from Unity menu
- iOS-only features guarded with `#if UNITY_IOS && !UNITY_EDITOR`
- When making database changes, update `Supabase/schema.sql` and run in both dev and prod
