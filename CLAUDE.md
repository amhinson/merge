# Overtone — Daily Merge Game

A daily puzzle-merge game built in Unity 6 (6000.3.11f1) with Supabase backend. Currently targeting iOS, with Android and web builds planned. Players drop balls, merge matching tiers, and compete on a daily leaderboard. Same seeded sequence for all players each day.

## Build & Deploy

### Supabase Edge Functions
```bash
cd Supabase && ./deploy.sh dev    # deploy to dev project
cd Supabase && ./deploy.sh prod   # deploy to prod project
```

### iOS
```bash
./build-ios.sh dev      # dev build → TestFlight
./build-ios.sh prod     # prod build → TestFlight
./build-ios.sh all      # both sequentially
```
Build number auto-increments from `.build-number`. Override with `BUILD_NUMBER=42 ./build-ios.sh prod`.

### Unity Scene
The gameplay screen is built at **editor time** by `GameSceneBuilder.cs`. After modifying it, run: **MergeGame > Build Game Scene** in the Unity menu, then save the scene (Cmd+S). Other screens (Home, Settings, Onboarding, Results) self-build at runtime via `OnEnable`.

## Architecture

### Namespaces
- `MergeGame.Core` — Game logic (GameManager, BallController, DropController, DailySeedManager, ScoreManager, MergeTracker, etc.)
- `MergeGame.UI` — All screens and HUD components (procedurally built, no prefabs)
- `MergeGame.Backend` — SupabaseClient, LeaderboardService
- `MergeGame.Data` — ScriptableObjects and design tokens (BallData, PhysicsConfig, OvertoneDesign)
- `MergeGame.Visual` — BallRenderer, WaveformAnimator, MergeParticles, PixelUIGenerator
- `MergeGame.Audio` — AudioManager (procedural SFX)
- `MergeGame.Editor` — GameSceneBuilder, BuildScript

### Key Patterns
- **Singletons** everywhere (GameManager.Instance, ScoreManager.Instance, etc.)
- **Callback-based async** for network calls (no async/await)
- **Coroutines** for animations, timing, and settle detection
- **ScriptableObjects** for ball tiers (BallData) and physics tuning (PhysicsConfig)
- **Procedural rendering** — ball sprites generated by BallRenderer, waveforms by shader

### Game Flow
```
App Launch → PreloadAndNavigate() → Check saved game?
  → Yes: ResumeGame() → Playing
  → No:  First launch? → Onboarding → HomeFresh
                       → HasPlayedToday? → HomePlayed / HomeFresh
Playing → Drop ball → Merge same tiers → Score with combo multiplier → Game Over → Results
```

### Daily Seed System
`DailySeedManager` generates a deterministic ball sequence from the date string hash (xorshift128). `sequenceIndex` tracks position. Same seed = same sequence for all players. Game state persistence saves/restores this index.

## Design System

### Constants (OvertoneDesign.cs)
- Colors: `OC.cyan` (#4DD9C0), `OC.bg` (#0F1117), `OC.surface` (#161B24), `OC.border` (#232838), `OC.muted` (white 22%), `OC.amber`, `OC.pink`
- Fonts: `OvertoneUI.PressStart2P` (headings/buttons), `OvertoneUI.DMMono` (body text)
- Button height: **44px** everywhere (home, settings, onboarding, modals)
- Primary buttons: `CreatePrimaryButton()` — cyan fill, scanlines, PressStart2P 11pt
- Ghost buttons: outline border + bg fill, PressStart2P 9pt, OC.muted text

### Ball Tiers (11 total, index 0-10)
Point values in `Assets/ScriptableObjects/BallData_Tier[1-11].asset`. Combo multiplier in `ScoreManager.cs`: `1.0 + (chainLength - 1) * 0.5`, capped at 4x.

## Supabase Backend

### Edge Functions (in `Supabase/supabase/functions/`)
| Function | Purpose |
|----------|---------|
| `register-player` | Upsert player, generate Phish-themed default name |
| `submit-score` | Insert daily score with merge_counts + longest_chain |
| `get-leaderboard` | Ranked player list for a game date |
| `get-player-rank` | Player's rank + total players |
| `get-player-profile` | Today's score, streaks, merge counts |
| `update-display-name` | Validate, sanitize, profanity check (obscenity lib) |
| `sync-streak` | Update streak counts |

### Shared
- `_shared/auth.ts` — CORS headers, API key verification
- `_shared/profanity.ts` — obscenity-based content filter

### Tables
- `players` (device_uuid PK, display_name, streaks)
- `daily_scores` (device_uuid + game_date unique, score, merge_counts, longest_chain)

### Dev vs Prod
`SupabaseClient.cs` auto-selects credentials: Editor/Development builds → dev project, Release builds → prod project. Bundle IDs: `com.overtone.game.dev` / `com.overtone.game`.

## Game State Persistence

Saved to PlayerPrefs after each drop settles + on app background. Includes: ball positions/tiers, score, sequence index, merge stats, shakes, attempt type. Discarded on day rollover or game over. Restored with balls frozen (kinematic → dynamic after 1 frame).

## Conventions
- No emoji in code or UI unless explicitly requested
- Prefer editing existing files over creating new ones
- All UI screens build themselves procedurally — no scene-based UI
- GameSceneBuilder changes require scene rebuild from Unity menu
- iOS-only features guarded with `#if UNITY_IOS && !UNITY_EDITOR`
